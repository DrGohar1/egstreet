## 2. إعداد Supabase من الصفر

### الخطوة 1: إنشاء مشروع

1. روح على **[supabase.com](https://supabase.com)** واضغط **Start your project**
2. سجّل دخول بـ GitHub
3. اضغط **New project**
4. اختار:
   - **Organization:** اسمك أو شركتك
   - **Project name:** `egstreet`
   - **Database Password:** اختار كلمة سر قوية واحفظها!
   - **Region:** `Central EU (Frankfurt)` أو `US East` (الأقرب لمصر)
5. اضغط **Create new project** وانتظر دقيقتين

---

### الخطوة 2: تشغيل الـ Database Schema

1. في Supabase، روح **SQL Editor** من القائمة الجانبية
2. اضغط **New query**
3. انسخ كل محتوى ملف `supabase/migrations/` بالترتيب وشغّله
4. أو اضغط **Run** على كل ملف migration بالترتيب التالي:

```
20260213131532_*.sql   ← الجداول الأساسية
20260213150507_*.sql   ← إعدادات إضافية
20260213153640_*.sql   ← أدوار المستخدمين
20260216082049_*.sql   ← نظام الإعلانات
20260217_000000_*.sql  ← المميزات الجديدة
20260411181608_*.sql   ← آخر تحديث
```

> ⚠️ **مهم:** شغّل الملفات بالترتيب التاريخي — ملف بعد ملف

---

### الخطوة 3: أخذ بيانات الاتصال

1. في Supabase، روح **Project Settings** (أيقونة التروس)
2. اضغط **API**
3. انسخ:
   - **Project URL** → (شبيه بـ `https://abcdefgh.supabase.co`)
   - **anon/public key** → (مفتاح طويل جداً يبدأ بـ `eyJ...`)

---

### الخطوة 4: إعداد Storage للصور

1. روح **Storage** في Supabase
2. اضغط **New bucket**
3. الاسم: `article-images`
4. فعّل **Public bucket** ✅
5. اضغط **Save**
6. كرر وعمل bucket تاني اسمه `avatars` (Public أيضاً)

## 3. تشغيل المشروع محلياً

### الخطوة 1: تحميل الكود

```bash
git clone https://github.com/DrGohar1/egstreet.git
cd egstreet
```

### الخطوة 2: تثبيت المكتبات

```bash
npm install
```

> ⏳ انتظر دقيقتين تقريباً — بيحمل ~800 مكتبة

### الخطوة 3: إعداد متغيرات البيئة

```bash
# انسخ ملف المثال
cp .env.example .env.local
```

افتح ملف `.env.local` وعدّل القيم:

```env
VITE_SUPABASE_URL=https://YOUR_PROJECT_ID.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=eyJhbGci...YOUR_ANON_KEY...
```

### الخطوة 4: تشغيل السيرفر

```bash
npm run dev
```

افتح المتصفح على: **http://localhost:5173**

## 4. إنشاء أول حساب Admin

### الخطوة 1: إنشاء الحساب

1. افتح الموقع على `http://localhost:5173/auth`
2. اضغط **إنشاء حساب جديد**
3. أدخل الإيميل وكلمة السر
4. تحقق من الإيميل (راجع صندوق البريد)

### الخطوة 2: منح صلاحية Super Admin

1. روح **Supabase → SQL Editor**
2. شغّل الأمر التالي (بعد ما تبدّل الـ UUID):

```sql
-- أولاً: اعرف الـ UUID بتاعك
SELECT id, email FROM auth.users WHERE email = 'your-email@example.com';

-- ثانياً: أضفه كـ super_admin
INSERT INTO public.user_roles (user_id, role)
VALUES ('paste-your-uuid-here', 'super_admin');
```

### الخطوة 3: الدخول للوحة التحكم

1. ارجع للموقع واضغط على صورتك أو الاسم
2. ستظهر زر **لوحة التحكم** أو روح لـ `/admin`

## 5. لوحة التحكم

### نظرة عامة

| الصفحة | الوصول | الوظيفة |
|--------|--------|---------|
| **Dashboard Overview** | `/admin` | إحصاءات سريعة — مقالات، مشاهدات، مستخدمين |
| **Article Management** | `/admin/articles` | كتابة، تعديل، نشر، حذف المقالات |
| **Category Management** | `/admin/categories` | إدارة الأقسام والأقسام الفرعية |
| **Breaking News** | `/admin/breaking` | تحديد الأخبار العاجلة وشريط Ticker |
| **User Management** | `/admin/users` | إدارة المستخدمين والأدوار |
| **Advertisement** | `/admin/ads` | إدارة الإعلانات في 5 مواقع |
| **Analytics** | `/admin/analytics` | تحليل المشاهدات والزيارات |
| **Comments** | `/admin/comments` | مراجعة التعليقات والإشراف |
| **Newsletter** | `/admin/subscribers` | قائمة المشتركين في النشرة البريدية |
| **AI Tools** | `/admin/ai-tools` | أدوات الذكاء الاصطناعي للمحتوى |
| **News Scraper** | `/admin/scraper` | جلب الأخبار تلقائياً |
| **Site Settings** | `/admin/settings` | إعدادات الموقع والـ Kill Switch |
| **Backup & Restore** | `/admin/backup` | نسخ احتياطي وإعادة الاستعادة |

---

### كتابة مقال جديد

1. روح **Article Management** → اضغط **مقال جديد**
2. أدخل:
   - **العنوان** — اختار عنوان جذاب
   - **الرابط (Slug)** — يتولد تلقائياً من العنوان
   - **المقتطف** — جملة أو جملتين للمعاينة
   - **الصورة الرئيسية** — ارفع من جهازك
   - **المحتوى** — باستخدام محرر TipTap الغني
   - **القسم والتاجات** — صنّف المقال
3. اختار:
   - ☑️ **خبر عاجل** — يظهر في شريط Breaking
   - ☑️ **مميّز** — يظهر في Hero الرئيسية
4. اضغط **نشر** أو **حفظ كمسودة**

---

### إضافة مواقع إعلانية

1. روح **Advertisement Management**
2. اضغط **إضافة إعلان جديد**
3. اختار الموضع:
   - `top` — أعلى الصفحة الرئيسية
   - `sidebar` — الشريط الجانبي
   - `bottom` — أسفل الصفحة
   - `article-top` — أعلى المقال
   - `article-bottom` — أسفل المقال
4. أدخل كود الإعلان (Google AdSense أو كود HTML)
5. حدد تاريخ الانتهاء (اختياري)

## 6. Kill Switch

### ما هو Kill Switch؟

نظام يخلّيك تفعّل أو توقف أي ميزة في الموقع **بدون أي تعديل في الكود** — من لوحة التحكم مباشرة.

### طريقة الاستخدام

1. روح **Site Settings → Feature Flags**
2. ستجد مفاتيح تشغيل/إيقاف لكل ميزة

### جدول المميزات

| المفتاح | الميزة | متى تقفله؟ |
|---------|--------|-----------|
| `feature_comments` | التعليقات | لو محتاج تراجع أو توقف مؤقتاً |
| `feature_newsletter` | النشرة البريدية | قبل ما تجهز القائمة البريدية |
| `feature_breaking_ticker` | شريط الأخبار العاجلة | وقت الصيانة |
| `feature_ads` | الإعلانات | **قفله وقت التسليم، افتحه بعد البيع** |
| `feature_ai_tools` | أدوات الذكاء الاصطناعي | **ميزة مدفوعة — قفلها للعرض** |
| `feature_news_scraper` | جلب الأخبار تلقائياً | **ميزة مدفوعة — قفلها للعرض** |
| `feature_vip_notifications` | إشعارات VIP | **ميزة مدفوعة — قفلها للعرض** |
| `feature_advanced_analytics` | تحليلات متقدمة | **ميزة مدفوعة — قفلها للعرض** |
| `feature_copy_protection` | حماية المحتوى من النسخ | حسب احتياجك |

### استراتيجية البيع المقترحة

```
📦 الباقة الأساسية (مجانية للعميل):
   ✅ نشر المقالات
   ✅ إدارة الأقسام
   ✅ تعليقات + نشرة بريدية
   ✅ لوحة تحكم كاملة

💎 الباقة الاحترافية (مدفوعة):
   🔒 أدوات AI للمحتوى
   🔒 News Scraper تلقائي
   🔒 إشعارات Push VIP
   🔒 تحليلات متقدمة
   🔒 تقارير مخصصة
```

## 7. Deployment

### النشر على Vercel (الأسهل والأسرع)

#### الطريقة 1: من GitHub (مباشرة)

1. روح **[vercel.com](https://vercel.com)** وسجّل دخول بـ GitHub
2. اضغط **New Project**
3. اختار **egstreet** من قائمة الريبوهات
4. في **Environment Variables** أضف:

```
VITE_SUPABASE_URL          = https://your-project.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY = eyJhbGci...
```

5. اضغط **Deploy** وانتظر دقيقتين
6. ستحصل على رابط مجاني مثل: `egstreet.vercel.app`

#### الطريقة 2: من Terminal

```bash
# تثبيت Vercel CLI
npm install -g vercel

# Build وDeploy
npm run build
vercel --prod
```

---

### ربط Domain مخصص

1. في Vercel → **Settings → Domains**
2. اضغط **Add Domain**
3. أدخل الدومين مثل `egstreet.com`
4. في مزود الدومين (Namecheap, GoDaddy, إلخ) أضف:
   - **CNAME Record:** `www` → `cname.vercel-dns.com`
   - **A Record:** `@` → `76.76.21.21`

## 9. الأسئلة الشائعة

### ❓ الصفحة بتظهر فارغة بدون مقالات

**السبب:** ما في مقالات منشورة في قاعدة البيانات بعد.
**الحل:** روح لوحة التحكم → Article Management → أضف مقال جديد واضغط نشر.

---

### ❓ Auth مش شغال — ما بيستقبلش تسجيل

**السبب:** إعدادات Supabase Auth تحتاج تفعيل.
**الحل:**
1. روح Supabase → Authentication → Settings
2. تأكد إن **Enable email confirmations** = Off (للتطوير)
3. أو فعّل SMTP لإرسال إيميلات حقيقية

---

### ❓ رفع الصور مش شغال

**السبب:** الـ Storage bucket غير موجود أو غير مضبوط.
**الحل:** تأكد إنك عملت bucket اسمه `article-images` وهو Public كما في الخطوة 4 من إعداد Supabase.

---

### ❓ الموقع بطيء على الاستضافة

**الحل:**
```bash
# Build محسّن للإنتاج
npm run build

# تحقق من حجم الـ Build
ls -lh dist/assets/*.js | head -10
```

إذا ملف JS أكبر من 500KB، تحتاج **Code Splitting** — تواصل معنا.

---

### ❓ لوحة التحكم مش بتفتح — يحولني للـ Home

**السبب:** مستخدمك مش عنده دور `super_admin` في جدول `user_roles`.
**الحل:** راجع الخطوة 4 — إنشاء أول حساب Admin.

---

### ❓ عايز أغير اسم الموقع أو اللوجو

**الملفات المطلوب تعديلها:**
```
src/components/Header.tsx          ← اللوجو في الهيدر
public/favicon.ico                 ← أيقونة التبويب
public/og-image.png                ← صورة المشاركة
supabase/Site Settings → site_name ← اسم الموقع في قاعدة البيانات
```
